## Combination Sum III (Java Edition)

Here is the complete Java implementation of the backtracking solution. In Java, we use a `List<Integer>` to act as our dynamic tracking array (the stack) and back-and-forth mutate it using `.add()` and `.remove()`.

### \## 1. Java Implementation

```java


    import java.util.ArrayList;
    import java.util.List;
    
    class Solution {
        public List<List<Integer>> combinationSum3(int k, int n) {
            List<List<Integer>> result = new ArrayList<>();
            // Start the backtracking process from number 1
            backtrack(1, k, n, new ArrayList<>(), result);
            return result;
        }
    
        private void backtrack(int start, int k, int remainingSum, List<Integer> currentCombination, List<List<Integer>> result) {
            // Base Case 1: If we have picked exactly k numbers
            if (currentCombination.size() == k) {
                // If the numbers sum up to the target, save a deep copy of the list
                if (remainingSum == 0) {
                    result.add(new ArrayList<>(currentCombination));
                }
                return;
            }
    
            // Base Case 2: Pruning optimization
            // If the remaining sum drops below 0, no need to explore further numbers
            if (remainingSum < 0) {
                return;
            }
    
            // Try every number from 'start' up to 9
            for (int i = start; i <= 9; i++) {
                // 1. Choose: Add the number to our path
                currentCombination.add(i);
    
                // 2. Explore: Move to the next number (i + 1) and update the remaining sum
                backtrack(i + 1, k, remainingSum - i, currentCombination, result);
    
                // 3. Un-choose (Backtrack): Remove the last element to try the next iteration branch
                currentCombination.remove(currentCombination.size() - 1);
            }
        }
    }
```

### \## 2. Critical Java Nuances to Remember

When implementing backtracking in Java, interviewers look out for these specific details:

-   **Deep Copying Results:** When adding a valid combination to `result`, you cannot simply write `result.add(currentCombination);`. Because Java passes objects by reference, you would end up storing a list of empty references (since the array is cleared completely at the end of recursion). You must create a fresh copy: `new ArrayList<>(currentCombination)`.

-   **Efficient Removals:** We use `currentCombination.remove(currentCombination.size() - 1)` because removing from the end of an `ArrayList` is an $\\mathcal{O}(1)$ operation.

| **Choose 3** | `[1, 3]` | $4 - 3 = 1$ | $4$ | Base Case hit! Size is $k=2$, but sum $\\neq 0$. _Backtrack!_ |
| **Choose 4** | `[1, 4]` | $4 - 4 = 0$ | $5$ | **Success!** Size is $k=2$, sum is $0$. Add to `result`. _Backtrack!_ |
| **Pop 1** | `[]` | $5$ | $2$ | Loop continues to $i = 2$ |
| **Choose 2** | `[2]` | $5 - 2 = 3$ | $3$ | Dig deeper... |
